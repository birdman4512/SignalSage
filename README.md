# SignalSage

A self-hosted threat intelligence bot for **Slack** and **Discord**. Drop an IP, domain, URL, hash or CVE into any monitored channel and get an enriched report back within seconds. Also runs a configurable scheduled digest that summarises RSS feeds and podcasts using a local LLM.

---

## Features

**IOC Enrichment**
- Automatically detects IPs, domains, URLs, file hashes, emails and CVEs in messages
- Queries multiple threat intel providers in parallel: VirusTotal, Shodan, GreyNoise, AbuseIPDB, OTX, URLhaus, ThreatFox, MalwareBazaar, IPInfo, CIRCL CVE, URLScan
- Returns a colour-coded card (green/red) with per-provider verdicts and a short LLM assessment
- Results cached for 1 hour to avoid burning API quota

**Scheduled Digest**
- Each topic runs on its own cron schedule and posts to its own channel
- Sources can be RSS/Atom feeds or plain web pages (auto-detected)
- Podcast RSS feeds with audio enclosures are transcribed via Whisper before summarisation
- LLM summary generated by local Ollama (default) or Anthropic Claude API
- Each digest includes:
  - 📌 **Top Signals** — 3–5 cross-source bullets synthesised by the LLM
  - Severity-sorted items (🔴 Critical / 🟠 High / 🟡 Medium / 🟢 Low) with direct article links
  - 🔥 **Trending** badge on items that appeared in previous digests within the past 7 days
  - 📡 Source coverage footer with count, 🟢/🟡/🔴 confidence rating, and warnings for empty or chronically-failing feeds
  - Cross-topic deduplication — items already seen in an earlier topic that day are silently dropped

**Bot Commands**
| Command | Action |
|---|---|
| `!digest` | Run all topics immediately |
| `!digest <tag>` | Run one topic by tag (e.g. `!digest cyber`, `!digest vuln`) |
| `!digest list` | Show all scheduled topics, tags and next run time |
| `!help` | Show command reference |

## Quick Start

### Prerequisites
- Docker + Docker Compose
- A Slack or Discord bot token (see [docs/setup.md](docs/setup.md))

### 1. Clone and configure

```bash
git clone https://github.com/birdman4512/SignalSage.git
cd SignalSage

cp config/config.example.yaml config/config.yaml
cp config/watchlist.example.yaml config/watchlist.yaml
cp .env.example .env
```

Edit `.env` with your tokens and API keys.

### 2. Start

```bash
docker compose up -d --build
docker compose logs -f
```

Ollama will pull the configured model automatically on first start (~2 GB, one-time download). The bot is ready once you see `SignalSage started` in the logs.

### 3. Test it

Invite the bot to a channel (`/invite @SignalSage` in Slack), then post:
```
8.8.8.8
```

## Configuration

### `config/config.yaml`
Controls which platforms are enabled, which channels to monitor, provider API keys, LLM backend, and digest scheduling defaults.

Key digest settings:

| Key | Default | Description |
|---|---|---|
| `digest.llm_provider` | `ollama` | `ollama` or `anthropic` |
| `digest.ollama_model` | `gemma2:2b` | Model to pull and use |
| `digest.max_chars_per_source` | `3000` | Characters fetched per source |
| `digest.max_total_chars_per_topic` | `20000` | Total prompt budget per topic |
| `digest.data_dir` | `data` | Path for history and source health files |

### `config/watchlist.yaml`
Defines digest topics — each with its own schedule, channel, short tags, and list of sources:

```yaml
topics:
  - name: "Vulnerability Alerts"
    tags: [vuln, cve]               # !digest vuln
    schedule: "0 7 * * 1-5"        # 7am weekdays
    lookback: "24h"                  # only items from the last 24 hours
    digest_channel: "#vuln-alerts"
    sources:
      - name: "CISA Advisories"
        url: "https://www.cisa.gov/cybersecurity-advisories/all.xml"
```

Tags must be unique across all topics — `!digest <tag>` returns on the first match.

Both files are gitignored — the committed `*.example.*` files serve as templates.

### LLM options

| Setting | Description |
|---|---|
| `LLM_PROVIDER=ollama` | Local, free. Ollama runs inside Docker automatically. |
| `LLM_PROVIDER=anthropic` | Paid Claude API. Set `ANTHROPIC_API_KEY`. No local GPU/RAM needed. |

Recommended local models (set `OLLAMA_MODEL` in `.env`):

| Model | RAM | Notes |
|---|---|---|
| `gemma2:2b` | ~2.1 GB | Default. Fast and reliable JSON output. |
| `phi3:mini` | ~2.3 GB | Good instruction following. |
| `qwen2.5:1.5b` | ~1.0 GB | Smallest viable option. |

## Architecture

```
Message → IOC extraction → parallel provider lookups → cached results
                                                      → LLM assessment (async update)
                                                      → Slack Block Kit card / Discord message

Cron schedule → fetch RSS/web/podcast sources → Whisper transcription (audio)
             → LLM summarisation (Top Signals TLDR + severity-sorted items)
             → trend classification vs 7-day history
             → cross-topic deduplication (within same day)
             → post to digest channel with source health footer
```

Digest history and source health are persisted in `./data/` (mounted as a Docker volume) so trend detection and failure tracking survive container restarts.

All inter-service communication uses a private internal Docker network. Only `signalsage` and `ollama` have outbound internet access.
